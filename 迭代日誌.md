# 迭代日誌

本 repo 由「大腦 agent」持續迭代：每輪派 3 個 sub agent 從不同視角找當前痛點 → 評估收斂 → commit & push。記錄每輪採納/否決的決策，直到擠不出更好的做法為止。

## 第 1 輪（2026-06-24）

**3 個 sub agent 視角**：① 檢索準確度技術缺口 ② prompt 可執行性 ③ 真實失敗模式/維運。

**採納（寫入 v2）**：
1. 🔴 資料治理前置（敏感資訊遮罩）→ 新增「階段 0a」+ 設計文件治理章節 A。JIRA 單含帳密/個資，補晚 corpus 要重洗。
2. 🔴 困難負例標註 + 「陷阱單擠進 top-1/3 比率」對抗指標 → 階段 0。只測 Recall@k 量不到「自信地找錯」。
3. 🔴 query/document 對稱萃取 + chunk 粒度上限 → 階段 2。「以問搜問」兩端萃取需對稱否則破功。
4. 🔴 失效解法處理（時效污染提前 + 人工失效黑名單）→ 階段 2 + 治理章節 B。系統越準、散播錯誤解法越快。
5. 🔴 回饋飛輪標註去偏（強/弱信號、位置偏差、否決分因）→ 階段 4 + 治理章節 C。
6. 🔴 驗收門檻無數字的自我矛盾 → 執行原則加「門檻產生機制 + `【待填】` 參數」。
7. 🟠 階段1門檻基於現有關鍵字分數、pipeline 最小判定規則、golden set 算術矛盾(270 vs 300-500)+缺料退路、順序權威聲明、漂移監控常設、延遲/成本 SLA + 可降級。

**否決（避免過度設計）**：HyDE / multi-query、knowledge graph、自建 fine-tune embedding。理由：第一版「文字類找相似單」範圍內屬邊際遞減或殺雞用牛刀；Agent 1 自己也標為不建議。

**收斂判斷**：未收斂。第一輪缺口多且高價值，3 視角互補幾無重複。續跑下一輪。
